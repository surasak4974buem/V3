ระบบควบคุมคลังอะไหล่แพทย์

ศูนย์เครื่องมือแพทย์ โรงพยาบาลนครพิงค์ — ระบบบริหารคลังอะไหล่ & แจ้งเตือน (SMM 07-1:2024)

🌐 surasak4974buem.github.io/V3 🔐 รหัสเริ่มต้น: NKP-admin-2026 📅 ฉบับ 16 มิ.ย. 2569

คู่มือนี้อธิบายขั้นตอนการใช้งานระบบควบคุมคลังอะไหล่เครื่องมือแพทย์ทั้ง 10 โมดูล ตั้งแต่การเข้าสู่ระบบ จนถึงการตั้งค่าเชื่อมต่อคลาวด์และ LINE โดยมีภาพประกอบจริงจากหน้าจอระบบในแต่ละขั้นตอน

1

การเข้าสู่ระบบ

เข้าใช้งานระบบด้วยรหัสผ่านความปลอดภัย

เปิดเว็บเบราว์เซอร์แล้วเข้าสู่ที่อยู่ของระบบ จากนั้นกรอกรหัสผ่านเพื่อยืนยันตัวตน ระบบรองรับ 3 ระดับสิทธิ์ (Admin / Stock Manager / General User)

  1. เปิดเบราว์เซอร์ไปที่ https://surasak4974buem.github.io/V3/
  2. คลิกที่ช่อง “ระบุรหัสผ่านเข้าใช้งาน” แล้วกรอกรหัสผ่าน (รหัสเริ่มต้นคือ NKP-admin-2026)
  3. กดปุ่ม “เข้าสู่ระบบ (Sign In)”
  4. เมื่อเข้าสำเร็จจะมีหน้าต่าง “เข้าสู่ระบบสำเร็จ” ขึ้นมา ให้กด “รับทราบ (Acknowledge)” เพื่อเริ่มใช้งาน
🔑
ควรเปลี่ยนรหัสผ่าน หลังเข้าใช้งานครั้งแรก ผ่านเมนู “เปลี่ยนรหัส” ที่มุมล่างซ้าย เพื่อความปลอดภัยของระบบ
2

แดชบอร์ดภาพรวม

วิเคราะห์สถิติคลังสินค้า ความปลอดภัย และอัตราค่าบำรุงรักษา

หน้าแรกหลังเข้าสู่ระบบ แสดงภาพรวมสถานะคลังแบบเรียลไทม์ ทั้งจำนวนรายการ ความเสี่ยงสต็อกต่ำ มูลค่าคลังรวม และกราฟวิเคราะห์

📦

การ์ดสรุปสถานะ

จำนวนรายการทั้งหมด, อะไหล่ต่ำกว่าจุดสั่งซื้อ, ใกล้หมดอายุ และมูลค่าคลังรวม

📊

กราฟสต็อกวิกฤต

เปรียบเทียบสต็อกสูงสุด / ขั้นต่ำ / จุดสั่งซื้อ / คงคลังจริง รายอะไหล่

🥧

การกระจาย ABC

สัดส่วนอะไหล่กลุ่ม A (มูลค่าสูง) / B / C ตามหลัก ABC Analysis

🔔

ตารางเร่งด่วน

รายการอะไหล่ที่ต้องดำเนินการ (ต่ำกว่าเกณฑ์ / หมดอายุ) พร้อมผู้จัดจำหน่าย

  1. ชี้ที่แท่งกราฟ เพื่อดูรายละเอียดอะไหล่แต่ละรายการ (สต็อกสูงสุด/ขั้นต่ำ/จุดสั่งซื้อ/คงคลังจริง)
  2. ใช้ปุ่ม ย่อ/ขยายเมนู (<) มุมบนซ้าย เพื่อขยายพื้นที่แสดงผล
  3. สลับ โหมดกลางคืน (Dark Theme) ได้จากสวิตช์มุมล่างซ้ายของเมนู
3

คลังอะไหล่และอุปกรณ์

จัดการรายชื่ออะไหล่ สเปกทางเทคนิค และจุดสั่งซื้อที่ได้รับการระบุ

ทะเบียนอะไหล่หลัก (Master Catalog) แสดงรายการทั้งหมดพร้อมรหัส ชื่อ แบรนด์/ผู้ผลิต ระดับ ABC ตำแหน่งจัดเก็บ คงคลัง ราคา และวันหมดอายุ

  1. ค้นหา อะไหล่ด้วยรหัส ชื่อ หรือผู้ผลิต ในช่องค้นหาด้านบน
  2. กรอง ตามกลุ่ม ABC หรือสถานะคลัง (ปกติ / ต่ำ / หมด) จาก dropdown
  3. กด “เพิ่มรายการอะไหล่ใหม่” เพื่อเพิ่มอะไหล่ พร้อมกำหนดสต็อกขั้นต่ำ จุดสั่งซื้อ และวันหมดอายุ
  4. ใช้ Export CSV / Import CSV เพื่อนำข้อมูลออก/เข้าเป็นชุด
🏷️
ป้าย Class A/B/C คำนวณอัตโนมัติจากมูลค่าคงคลัง (ราคา × จำนวน) เพื่อจัดลำดับความสำคัญในการบริหารสต็อก
4

เบิกจ่ายและรับเข้าคลัง

ทำกิจกรรม เบิก จ่าย รับของ ยืม คืน และตรวจสอบความถูกต้องของระบบคลัง

บันทึกการเคลื่อนไหวของอะไหล่ทุกประเภท ระบบบังคับกรอกซีเรียลเครื่องมือแพทย์และเลขใบแจ้งซ่อมเพื่อให้ตรวจสอบย้อนกลับได้ครบถ้วน

  1. เลือก ประเภทรายการ: เบิกจ่ายไปซ่อมบำรุง (Issue) / รับเข้าคลัง (Receive) / ยืม-คืน
  2. เลือก อะไหล่ทางการแพทย์ จากรายการ (แสดงจำนวนคงเหลือกำกับ)
  3. กรอก จำนวน, ซีเรียลเครื่องมือแพทย์ (เช่น MEC-MON-001) และ ประเภทงานซ่อม (CM/PM)
  4. ระบุ เลขใบแจ้งซ่อม (6 หลัก) และ หน่วยงานที่เบิก (เช่น OPD, ICU, MEC)
  5. กดบันทึก — ระบบจะตัด/เพิ่มสต็อกและส่งแจ้งเตือน LINE ตามสถานะคงคลัง
⚠️
ช่องที่มีเครื่องหมาย * เป็นข้อมูลบังคับ — โดยเฉพาะ ซีเรียลเครื่อง และ เลขใบแจ้งซ่อม เพื่อความถูกต้องในการตรวจสอบย้อนหลัง
5

ประวัติคลัง (Ledger Logs)

ตรวจสอบประวัติการเบิกจ่าย รับเข้า ยืมคืน อะไหล่สำรองคลังทั้งหมด

สมุดบัญชีคลัง (Ledger) บันทึกทุกธุรกรรมพร้อมวันเวลา ประเภท อะไหล่ จำนวน เลขเครื่อง/งาน ผู้ปฏิบัติงาน และสถานะการยืม

  1. ค้นหา ด้วยรหัสอะไหล่ ชื่ออะไหล่ ผู้ปฏิบัติงาน หรือเลขงาน
  2. กรอง ตามประเภทรายการ และช่วงเวลา (รายวัน/สัปดาห์/เดือน/ปีงบ)
  3. กด “สรุปรายงานประจำเดือน” เพื่อสร้างรายงานการใช้อะไหล่รายเดือน
  4. กด Export CSV เพื่อนำประวัติออกเป็นไฟล์
6

แผนบำรุงรักษาเชิงรุก (PM)

ตาราง PM ประจำปี และคาดการณ์ความต้องการวัสดุอะไหล่ในอนาคต

วางแผนบำรุงรักษาเชิงป้องกัน (Preventive Maintenance) พร้อมระบุอะไหล่ที่ต้องใช้ในแต่ละแผน และระบบคาดการณ์การเติมอะไหล่ล่วงหน้า

  1. กด “เพิ่มแผน PM” เพื่อสร้างแผนใหม่ พร้อมระบุรุ่นเครื่อง ความถี่ และอะไหล่ที่ต้องใช้
  2. ดู วันกำหนดทำ และช่างผู้รับผิดชอบในตาราง
  3. เลือก ช่วงเวลาคาดการณ์ล่วงหน้า (เช่น 90 วัน) แล้วกดคำนวณ เพื่อดูความต้องการอะไหล่ล่วงหน้า
  4. ตรวจ รายงานอะไหล่ขาดแคลนตามแผน PM (Shortage Report) เพื่อเตรียมจัดซื้อ
7

ใบขอจัดซื้ออะไหล่

ตรวจสอบชิ้นส่วนที่หมดเกณฑ์สำรอง และสร้างใบขอเสนอซื้อแบบอัตโนมัติ

ระบบแจ้งเตือนและวิเคราะห์จุดสั่งซื้อ (Reorder Point Monitor) แสดงอะไหล่ที่คงคลังถึง/ต่ำกว่าจุดสั่งซื้อ พร้อมประมาณการมูลค่าและปุ่มส่งเข้าข้อความ LINE

  1. ตรวจรายการอะไหล่ที่ คงคลังถึง/ต่ำกว่าจุดสั่งซื้อ (ไฮไลต์สถานะ)
  2. ดู จำนวนที่ต้องสั่งซื้อ และ ประมาณการมูลค่า ที่ระบบคำนวณให้
  3. กด “เพิ่มลงในบันทึกข้อความ” เพื่อรวมเข้าบันทึกข้อความขอจัดซื้อ
  4. นำรายการไปออก บันทึกข้อความ/ใบขอซื้อ เพื่อพิมพ์เสนอผู้บริหาร
8

แผนงบประมาณประจำปี

ติดตามวงเงิน ใช้ไปแล้ว คงเหลือ และจัดการรายการงบประมาณแต่ละโครงการ

แดชบอร์ดงบประมาณรายปี เลือกดูตามปีงบประมาณได้ แสดงวงเงินรวม ยอดใช้ไป คงเหลือ ความคืบหน้ารายโครงการ และกราฟเปรียบเทียบงบ vs ใช้ไป

  1. เลือก ปีงบประมาณ จาก dropdown มุมขวาบน (หรือ “ทุกปีงบประมาณ” เพื่อดูรวม)
  2. ดู การ์ดสรุป: วงเงินงบประมาณทั้งหมด, ใช้ไปแล้ว, คงเหลือ และสัดส่วน %
  3. ติดตาม ความคืบหน้ารายโครงการ ผ่าน progress bar รวมและแยกรายหมวด
  4. กด “เพิ่มรายการงบ” (เฉพาะ Admin) เพื่อเพิ่ม/แก้ไขวงเงินแต่ละโครงการ
🤖
หมวดหลัก 4 หมวดที่มีป้าย AUTO ระบบจะคำนวณ “ใช้ไปแล้ว” อัตโนมัติจากการเบิกจ่ายจริง ส่วนรายการอื่นกรอกยอดใช้เองได้ในหน้าจัดการ
👁️
สิทธิ์: Admin จัดการ (เพิ่ม/แก้ไข/ลบ) ได้เต็ม ส่วน Stock Manager ดูได้อย่างเดียว (มีป้าย “ดูได้อย่างเดียว”)
9

ระบบจัดการสิทธิ์ผู้ใช้

จัดการผู้ใช้ระบบและกำหนดรหัสผ่านแยกสิทธิ์ในการใช้งาน

สร้างและบริหารบัญชีผู้ใช้ พร้อมกำหนดระดับสิทธิ์ 3 ระดับ และดูจำนวนครั้งการเข้าใช้ระบบของแต่ละบัญชี

ระดับสิทธิ์สิ่งที่ทำได้โควต้า
Adminจัดการได้ทุกโมดูล รวมงบประมาณ ผู้ใช้ และการตั้งค่าคลาวด์/LINEสูงสุด 3
Stock Managerจัดการคลัง เบิกจ่าย PM ได้เต็ม — ดูงบประมาณได้อย่างเดียวสูงสุด 3
General Userดูข้อมูลภาพรวมและประวัติ (จำกัดการแก้ไข)สูงสุด 10
  1. กรอก ชื่อ-นามสกุลจริง และเลือก ระดับสิทธิ์การใช้งาน
  2. กำหนด รหัสผ่าน เอง หรือกด “สุ่มรหัส”
  3. กด “บันทึกข้อมูลผู้ใช้” — บัญชีใหม่จะปรากฏในตารางด้านขวา
  4. ใช้ปุ่ม แก้ไข / ลบ และสลับ สถานะเปิด-ปิดใช้งาน รายบัญชีได้
10

คลาวด์ & LINE Settings

ตั้งค่าการเชื่อมต่อฐานข้อมูล Google Sheets และระบบ LINE OA Notifications

เชื่อมต่อระบบกับ Google Sheets เพื่อสำรอง/ซิงก์ข้อมูลบนคลาวด์ และตั้งค่า LINE OA เพื่อรับการแจ้งเตือนอัตโนมัติ (เฉพาะสิทธิ์ Admin)

  1. วาง Google Apps Script Web App URL (ฐานข้อมูลคลาวด์ซิงก์)
  2. เปิดสวิตช์ “ส่งแจ้งเตือน LINE OA เมื่อแตะนีออน” และวาง LINE Channel Access Token + LINE Group/User ID
  3. กด “บันทึกการตั้งค่า” แล้วกด “ทดสอบส่งข้อความ LINE” เพื่อตรวจสอบการเชื่อมต่อ
  4. ทำตาม ขั้นตอนติดตั้ง ด้านขวา: สร้าง Google Sheet → วางโค้ด Apps Script (ปุ่ม Copy Code) → Deploy เป็น Web App (สิทธิ์ “ทุกคน/Anyone”)
🔒
ความปลอดภัยสำคัญ: LINE Channel Access Token เก็บไว้ในเครื่องของผู้ใช้เท่านั้น ไม่ถูกอัปโหลดหรือบันทึกลงคลังโค้ดสาธารณะ (GitHub) เด็ดขาด
☁️
เมื่อเชื่อมต่อสำเร็จ ข้อมูลคลัง อะไหล่ ผู้ใช้ แผน PM และ แผนงบประมาณ (BudgetPlan) จะถูกซิงก์ขึ้น Google Sheets อัตโนมัติทุกครั้งที่บันทึก
